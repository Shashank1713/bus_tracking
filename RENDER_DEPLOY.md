# Deploy This Project on Render

This project runs as a single Node web service on Render (backend APIs + static frontend from `public/`).

## 1. Push to GitHub
Push this repository to GitHub/GitLab so Render can access it.

## 2. Create Service from Blueprint
In Render, use **New +** -> **Blueprint** and select this repo.
Render will read `render.yaml` and create the `bus-tracking` web service.

## 3. Set Required Environment Variables
In Render service settings, set these before first start:

- `MONGO_URL` (required): MongoDB connection string (Atlas or other hosted MongoDB)
- `PUBLIC_BASE_URL` (recommended): your Render URL, e.g. `https://bus-tracking.onrender.com`

`JWT_SECRET` is auto-generated by `render.yaml`.

## 4. Optional Environment Variables
Set only if you use these features:

- Email OTP: `GMAIL_USER`, `GMAIL_APP_PASSWORD`
- SMS OTP (FAST2SMS): `FAST2SMS_API_KEY`, `FAST2SMS_SENDER_ID`, `FAST2SMS_TEMPLATE_ID`
- Firebase Admin login: `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY`
- Firebase JSON alternative: `FIREBASE_SERVICE_ACCOUNT_KEY`
- Firebase web config for client: `FIREBASE_WEB_API_KEY`, `FIREBASE_WEB_AUTH_DOMAIN`, `FIREBASE_WEB_PROJECT_ID`, `FIREBASE_WEB_APP_ID`

## 5. Deploy + Verify
After deploy succeeds:

- App: `/login.html`
- API health: `/api/openapi.json`
- Swagger UI: `/api-docs`

## 6. Seed Demo Data (Optional)
Call:

```bash
curl -X POST https://<your-render-domain>/api/admin/seed-demo
```

This creates demo routes and trips for testing search and booking flows.
